How to read this report

Every row below is one link Zvi included in a post. Rows are grouped into branches by the publisher's domain (who put it out), not by topic, and sorted newest first. The middle column is Zvi's own wording for the link, captured verbatim — it is not the title of the destination page, so a name or a short phrase can point to a differently-titled page. Where his wording is too generic or too brief to be useful (e.g. “here”, “stated”, or a bare name), the row instead shows a title built from the link’s own URL slug, not from the page behind it. When Zvi links the same URL more than once, those rows merge into one with a ×N badge (the branch’s link count still tallies every citation), and kept wording is lightly tidied for display (leading capital, trailing punctuation trimmed). The left column always shows the domain, so you can see where a link goes before clicking. For ambiguous links (videos, one-offs, or generic wording like “here”), a collapsible context toggle reveals the verbatim paragraph the link sits in, with Zvi’s link words highlighted, so you can judge how it relates without leaving the page.

How to read this

Momentum is how often each name is mentioned across posts in the chosen window (This run, or the last 7 / 30 / 60 / 90 days / 6 months). The number on each pill is that count.

This run vs day windows. This run counts every post in the feed right now, with no date filter. The day windows count by post date, measured back from the latest post (so the same feed gives the same window totals whenever you run it). They can differ from This run: if the feed spans more days than the window, the oldest posts fall outside it, so This run sits above any window narrower than the feed. A shorter window is always a subset of a longer one, so a busy recent week can be a large share of the month.

Post count vs pill count. The post number under the dropdown is the size of the basket (how many posts the view covers); a pill’s number is how many times that name appears inside that basket. They move independently. Dropping a post that never mentions a name lowers the post count but not that name’s tally, so you can see, for example, 20 posts on This run and 19 in Last 30 days while a name stays on the same number, because the post that fell outside the 30-day window did not mention it.

The line under the dropdown shows how many posts the current counts cover, the window, and the latest post date they are measured to (e.g. "7 posts · Last 7 days · on 19 Jun"). In a window view, Movers lists the names with the biggest rise versus the equal window before it.

Trend (window views only) compares the chosen window with the equal window right before it: ▲ rising = mentioned more than the previous period, ▼ falling = fewer, – flat = the same. Next to the arrow, a small sparkline traces that same name across the last few equal windows (oldest on the left, newest on the right, ending at the current count), so you see the path and not just the latest step; it is coloured to match the arrow. It fills in as more history is collected. risingfallingflatA dashed pill is a name mentioned in the window but not cited in the current run. This run has no earlier window to compare against, so it shows counts only. The longest windows can also show counts only once they reach the start of stored history, since there is no earlier period left to compare.
